# Documentation Guidelines

Follow these guidelines when creating or updating documentation:

## Structure

- Start with a clear, concise title
- Include a brief description (1-2 sentences) immediately after the title
- Use hierarchical headings (H2 for main sections, H3 for subsections)
- Keep paragraphs short (3-5 sentences)

## Content

- Write for your target audience
- Define technical terms on first use
- Include practical examples
- Add code snippets when relevant
- Link to related documentation

## Formatting

- Use bullet points for lists of 3+ items
- Use numbered lists for sequential steps
- Use code blocks with language specification
- Add alt text to images
- Use tables for comparing options

## Maintenance

- Review documentation with each release
- Remove outdated information promptly
- Update examples when APIs change
- Check and fix broken links regularly
